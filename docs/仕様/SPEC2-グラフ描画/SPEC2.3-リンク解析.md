# SPEC2.3: リンク解析エンジン

[REQ2.1 — グラフ描画](../../要件定義/REQ2-リンクグラフ/REQ2.1-グラフ描画.md) の「マークダウンリンク + 自動マッチ」を解決するバックエンド実装。

## 設計

- 実装: `src/lib/links.ts`
- 入力: `FileRef[]`（全 Markdown ファイルの path と displayName）と readFile
- 出力: `{ nodes, edges }`。参照されなかったファイルは nodes から除外される

### リンク抽出の 2 系統

1. **明示的マークダウンリンク**: `[label](url)` を正規表現 `/(^|[^!])\[([^\]]*)\]\(([^)]+)\)/g` で抽出（先頭の `!` を除外して画像記法は対象外）
2. **displayName の自動マッチ**: 本文中に現れるファイル displayName（`stripOrderPrefix` 後の名前）を長い順にソートし、1 本の正規表現で全文走査。displayName の長さが 2 文字以下のものは対象外

### 前処理

- フェンスコードブロック（\`\`\` / \~\~\~）とインラインコードは走査前にストリップし、自動マッチの誤爆を防ぐ
- 明示的リンクを解決した後に、URL 部分 `(url)` を除去してから自動マッチを走らせる。URL 中のフォルダ名が displayName と偶然一致してもエッジを張らないようにするため

### resolveLink

- `#`・`?` 以降を除去
- `scheme:` 形式や `//` 始まりは外部リンクとして `null`
- 相対パスを `currentPath` のディレクトリ基準で解決し、`.` `..` を正規化
- 候補の順: 完全一致 → `+ "/README.md"` → `+ ".md"`
- いずれも `fileSet` に無ければ `null`

### キャッシュ

- `getCachedLinkGraph()` は `(path, mtime, displayName)` のタプル集合をシグネチャ化し、無変更ならメモリキャッシュを再利用
- 変更検知時のみ `buildLinkGraph()` を再実行
